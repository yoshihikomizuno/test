# new-project — 新規案件のキックオフ

新しい制作案件を、この `test` リポジトリ内の**新しいフォルダ**として立ち上げ、開発を始めるためのスキル。
「同じリポジトリにフォルダを足す」形なので、Cloudflareの追加設定なしで自動プレビューされる。

## 発動する言葉（例）
「新しい開発環境を作って」「デモサイトを作って」「新規案件フォルダを作って」「新しい案件を始めたい」「新しいサイトを作りたい」など。

## 手順

### STEP 0 — まず不明点を確認する（最重要）

**いきなり作り始めない。** ユーザーのメッセージで未確定の項目を、`AskUserQuestion` などで確認する。
すでに指定済みの項目は聞き返さない。妥当な**デフォルト案を提示して「これでよいか」**を確認する形にする。

確認する項目：
1. **何を作るか**（サイトの種類・目的・主な内容）— 最重要。例：LP／デモ／社内ツール など
2. **フォルダ名**（URLに使う。半角英数とハイフンのみ。例 `abc-lp`）— こちらで案を出して確認
3. **表示名**（一覧カードと `<title>` に使う日本語名）
4. **デザインの方向性**（マザリアルの世界観に寄せる？／参考URL・画像・配色・トーン。任意）
5. **公開/非公開**（既定は「公開」。社内限定にするなら後で Access 設定）
6. **納期・優先度**（任意）

不明点が1つでもあれば、着手前に確認する。ユーザーが「おまかせ」なら、デフォルト（フォルダ名は内容から生成／マザリアルの世界観／公開）で進めてよい。

### STEP 1 — フォルダを作成

確認が取れたら、雛形を作る：
```bash
bash scripts/new-preview-project.sh <フォルダ名> "<表示名>"
```
（`index.html` / `css/style.css` / `js/main.js` の雛形が生成される）

### STEP 2 — 一覧ダッシュボードにカードを追加

ルートの `index.html` の `<main class="list">` に、既存の `<article>`(=`.card`) を1枚複製して、
新案件用にリンク先(`./<フォルダ名>/`)・絵文字・タイトル・説明・パス表記を差し替える。

### STEP 3 — 制作（作業ブランチで）

- 作業ブランチで進める（main には直接入れない）。
- 実装後、可能なら Chromium でレンダリング確認（`run` スキル等）。
- コミットして push。

### STEP 4 — プレビューURLを伝える

push後、**ブランチプレビューURL**を案内する：
`https://<ブランチ名>-test.mazareal.workers.dev/<フォルダ名>/`
（mainにマージしなくても、このURLで確認できる。確定したら main にマージ → 本番URL）

## 参照
- デプロイ／プレビュー／公開・非公開の仕組みは **preview-deploy スキル**（`.claude/skills/preview-deploy/SKILL.md`）に従う。
- 既定はすべて公開。特定フォルダだけ非公開にしたい場合は preview-deploy の「公開/非公開の管理」を参照し、
  Cloudflare Access の手順をユーザーに案内する（Cloudflare操作はClaude側からは不可）。

## 注意
- フォルダ名は後からURLに残るので、確定前に必ずユーザー確認する。
- 秘密情報（APIキー等）は案件フォルダに置かない。
