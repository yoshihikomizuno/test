# lm（Log Memo）— GitHub/Web 環境向け

ローカルの `/lm`（handoff 3世代ローテーション＋エピソードログ＋Google同期）を、この環境の制約に合わせて翻案したもの。

## 環境メモ（なぜ翻案か）
- ローカル版は `D:/3aidmz@gmail.com/work/…` のローカルパスに書くが、Webセッションからは見えない。
- この環境の Google Drive 連携は「読み取り・新規作成」は可能だが「既存ファイルの更新（追記）」ができない。
  → **更新が必要な handoff はリポジトリ内（`handoffs/`）に置く**（git で追記・世代管理・push できる）。
  → Drive へは「当日サマリを1枚新規作成」する形でのみ連携（追記はしない）。

## トリガー
`/lm` / 「ログに残して」/「作業のまとめをhandoffに」/「今回の作業履歴を残して」/「引き継ぎメモを更新して」/「今日はここまで」「一旦区切り」

## 実行フロー

### Step 0: 次セッション着手プロンプト保存の確認（必須・最初に実行）
他の操作の前に AskUserQuestion で確認する：
- 質問「次セッション着手プロンプトを handoff に保存しますか？」
- 選択肢「Yes（推奨）：handoff 冒頭の固定セクションに保存（無ければ作成・あれば上書き）」/「No：handoff 本体のみ更新」
結果を OUTPUT_NEXT_SESSION_PROMPT(true/false) として保持。ユーザーが「不要」と明言済みなら確認省略で No。

### Step 1: 記録先の判定
- **特定の案件フォルダ（リポジトリ直下のフォルダ。例 `mazareal-kids-game/`）が主作業** → `handoffs/<フォルダ名>_handoff.md`（3世代ローテーション）
- **横断作業（スキル改修・プレビュー基盤・リポジトリ運用など、特定案件に属さない）** → `handoffs/GitHub_handoff.md`（append-only：古い記録を消さず新しいものを上に積む）
- 迷うときのみユーザーに1回だけ確認する。

### Step 2〜4: handoff の作成・更新
`handoffs/` が無ければ作成。対象ファイルを Read してから：
- **プロジェクト handoff**：3世代ローテーション（世代1→2→3、旧世代3は削除、新しい「世代1（最新）」を最上部に挿入）。
- **GitHub_handoff.md（横断）**：append-only。「## 記録（新しいものを上に追記）」直下に新エントリを積む。世代削除はしない。

各記録に必ず含める項目：
1. 意図・目的（背景と動機）
2. 採用した全体像（構成・技術）
3. 実施した作業（表 or 箇条書き）
4. 詰まった点と解決（問題→原因→解決。再現できる粒度で）
5. 使用したスキル・ツール
6. 進捗状況（✅/⏭/⚠️）
7. 次に取り掛かること（優先順・具体的に）
8. 運用上の注意
9. 関連ファイル（パス）

### Step 5: 次セッション着手プロンプト（Step 0 が Yes のときのみ）
handoff 冒頭の固定セクション「## 🚀 次セッション着手プロンプト（最新版・常に上書き）」に、コピペで使える着手プロンプトを保存（世代ローテーションの外・常に上書き）。読み込むべきファイル、現在地サマリ、次にやること、暗黙の前提、環境情報を含める。ターミナルには出さない。

### Step 6: コミット & push
`handoffs/` の変更を commit（例 `chore(handoff): <対象> セッション記録`）して designated ブランチに push。オープンなPRがあればそこに含める。

### Step 7（任意）: Google ドライブ work フォルダへ当日サマリを作成
Drive 連携が使える場合、`work` フォルダ（title='work' のフォルダを検索。既知ID: 1fcsQN_fJk_vpWGgTIl1jUpd7pbZhWpqe）に、当日のセッションサマリを Google ドキュメントとして**新規作成**（タイトル例「作業ログ YYYY-MM-DD <対象>」）。※Drive上のファイル更新は不可のため、追記ではなく毎回新規1枚。ローカルの `06_EPISODE_LOG.md`／`HANDOFF_INDEX.md` への追記・upsert は Web からは行えないため、ここでは代替として当日Docのみ。

### Step 8: 完了報告
更新した handoff パス／追加/削除した世代／次セッションプロンプトの保存先／Drive Doc の作成結果／次に見るべきファイル、を報告。

## 除外・やらないこと
- APIキー・トークン等の機密情報は書かない。
- コードの diff は貼らない（git history で追える）。
- 「いい感じ」等の曖昧表現は使わず、次セッションで判断できる具体性を担保。
- プロジェクト handoff は4世代以上残さない。GitHub_handoff.md（横断）は append-only で消さない。
